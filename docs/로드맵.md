## 0. 한 줄 결론

**문서가 부족해서 못 나가는 게 아니라, 문서는 게시 가능 수준인데 "채널"이 없어 못 나가는 상태.**
모집 키트·지원폼 문항·응대 템플릿·사이트 카피 9페이지가 완비되어 있고, 막고 있는 것은 ① 플레이스홀더 5개(인스타·폼 링크·연락처·주소·운영시간), ② 실제 접수 채널 부재, ③ 미검토 법률 초안의 공개 노출 리스크입니다.

따라서 **첫 작가 지원서가 실제로 도착하는 최단 퍼널(채널→폼→응대)을 먼저 열되**, 되돌릴 수 없는 것(핸들·도메인 선점, 정직성 패치, 초안 노출 통제)만 게시 전에 끝내고, 법무 검토는 지원 접수~입점 계약 사이의 리드타임에 병행합니다.

## 1. 현재 상태 진단

### 강점 (이미 완성된 자산)
- **개념·브랜드 정합**: 복지수수료 구조, 3단계(위탁→너투사→나투사), 용어 표준, 보이스&톤([`작가-소개`](작가-소개.md) · [`브랜드-정의서`](브랜드-정의서.md)).
- **사이트 카피 완비**: [`홈페이지-사이트맵`](홈페이지-사이트맵.md) v5.0 + [`홈-랜딩-카피`](홈-랜딩-카피.md) 11블록 + [`pages/`](pages/) 6종 — 예시/확정 구분 표기와 교체 지침 포함.
- **모집 키트 완성**: [`작가-모집-키트`](작가-모집-키트.md) — 플레이스홀더만 채우면 게시 가능.
- **기술 인프라 단순·건강**: 문서 → 정적 빌드 → GitHub Pages 자동 배포.

### 병목
| 병목 | 내용 |

### 발견된 문서 모순 (모집 전 정합화 필요)
1. **누진 복지수수료의 축이 문서마다 다름** — [`작가-소개`](작가-소개.md)는 *낸 제품 수*(1~3/4~10/11~), [`작가-운영정책`](작가-운영정책.md)·[`입점-동의서`](입점-동의서.md)는 *점유 칸*(1칸/2~3칸/4칸~), [`사업계획서`](사업계획서.md)는 병기. **개념 정본과 계약 정본이 서로 다른 단위 사용.**
2. **일반수수료 30%의 지위** — 운영정책은 확정처럼, 입점-동의서는 '예시·조정 가능'.
3. **복지수수료 인하 조건 서술 4갈래** — '매출 게이트'(홈) / '점유량 누진'(합류하기) / '잘 팔릴수록'(손님-안내) / 결합(문의 Q10).
4. **한옥 확보 상태 표기 불일치** — [`문서-지도`](문서-지도.md) §3은 '100평대·월세 500만+'를 확정 사실로, 사업계획서 로드맵 0단계는 '한옥 임대(TBD)'를 미래 과제로.
5. **상태 라벨 모순** — 브랜드-정의서 '✅ 확정' 배지 vs 본문 'v0.1 초안' 등.

## 2. 추천 로드맵

### Phase 0 — 기반 정리 *(즉시 · 반나절 · 전부 셀프)*
| # | 작업 | 왜 먼저 |
| --- | --- | --- |
| 0-1 | 진행 중인 뷰어 성능 최적화 PR 머지 → 이후 작업의 기준을 `main`으로 통일 | 작업 브랜치 분기 방지 |
| 0-2 | **미검토 법률 초안 노출 통제** — 입점-동의서·legal 3종에 "미검토 초안·법적 효력 없음" 배너 삽입(또는 뷰어 목록 제외) | 예비 작가가 초안을 확정 조건으로 오독하는 창을 0으로 — **채널 개설보다 먼저** |
| 0-3 | 배포 워크플로에 빌드 단계 추가(`build.py` 자동 실행) + 이미지 워크플로 `sources.json` 커밋 누락 수정 | 이후 "정합화 커밋 = 즉시 배포"가 보장되어야 낡은 조건 노출 재발 방지 |

### Phase 2 — 모집 사이트 MVP *(약 1주 · 예산 0원)*
- [`홈-랜딩-카피`](홈-랜딩-카피.md) 11블록 + 1차 5페이지(홈·소개·공간·운영정책·합류)를 **정적 사이트로 구축** — 기존 Python 빌드 방식 재사용.
- `/policy` 웹용 카피 신설([`작가-운영정책`](작가-운영정책.md) 요약) — **렌더 시 '예시' 라벨 보존**을 명시 요구사항으로.
- **최소 디자인 토큰** — [`브랜드-정의서`](브랜드-정의서.md) §8 무드(한지 아이보리·먹빛·단청 적·청자) 기반. *문서-지도 §5의 1순위 다음 작업.*
- 이미지: 한옥 확보 전엔 기존 SVG + 헐버트 박사 퍼블릭 도메인 사진 · **사진-촬영-리스트 문서화**(한옥 의존 컷 vs 즉시 수급 컷 구분).

### Phase 3 — 게시·응대·측정: 모집 개시 *(사이트 완성 직후)*
- **게시 전 체크리스트 문서화·실행** — 플레이스홀더 0개 · 30% 조건 명시 · 한옥 상태와 공고 표현 정렬 · 비제휴 표시 · 초안 노출 통제 확인. **통과 기록을 남김.**
- 대표 이미지·카드뉴스 제작 → 인스타 첫 게시 → 작가 커뮤니티·카페 공유 (모집 키트 §6).
- **제출 후 경험** — 폼 완료 화면에 응답 기한(예: 3일 내), 접수 알림, 응대 템플릿 매크로화.
- **KPI 기록 게시 당일 가동** — 채널별 유입은 소급 수집 불가. 지원 수·입점 수·채널별 유입 시트 1장 ([`성과지표-KPI`](성과지표-KPI.md) 연결).
- 예시 콘텐츠 정리 — 가상 작가 카드 4개·가상 게시물 2개 → "첫 동료를 기다립니다" 빈 상태 카피로 교체.

### Phase 4 — 지원→계약 전환 준비 *(Phase 3과 병행 · 첫 입점 전 완료)*
- **전문가 질의서 4종 작성 후 병렬 발주** — 변호사([`입점-동의서`](입점-동의서.md)·[`legal/`](legal/) 3종·게이트 재량 조항의 약관규제법 쟁점) / 변리사(상표 출원) / 노무사(너투사 위장도급) / 세무사(위탁 세무·통신판매 신고). *백지가 아닌 쟁점 목록을 들고 가야 자문이 1회에 끝남.*
- **개인정보 동의 문구만 소형 선행 검토 분리 발주** 옵션 — 변호사 전체 검토(2~4주)가 폼 운영의 단일 병목이 되지 않게.
- 입점-동의서 v0.2 셀프 보강 — 공란 안내화 · '현저한 부진' 객관 지표와 이의 절차 · 규칙 변경 시 사전 통지·이의권 조항.
- **게이트 산정 원칙 공개 초안** — '일정 매출' 기준을 내부 재량으로 두더라도 산정 방식(고정비 회수율 기반)과 변경 시 통지 절차를 공개 — 핵심가치 '투명'과의 긴장 해소.
- 상태 라벨 3단계 규칙(구조확정/수치확정/실데이터확정)을 [`문서-지도`](문서-지도.md)에 명문화 + 라벨 모순 일괄 수정.
- **모집 이후**: 나투사 거버넌스 설계(동료가 생긴 뒤 — [`정관-거버넌스`](정관-거버넌스.md)) · 작품 보험 견적 비교 · 운영 예시 수치의 실측 확정(2~3개월 데이터).

## 4. 점검에서 드러난 사각지대 (로드맵 밖 — 별도 트랙)

1. **초기 자본 계획 부재** — 보증금·권리금·인테리어·집기 등 **개점 일시금** 산정이 어느 문서에도 없음(월 고정비만 논의) → 산정 틀과 단계적 개관 시나리오를 [`시장-벤치마크-개관-전략`](시장-벤치마크-개관-전략.md)에 마련, [`사업계획서`](사업계획서.md) 편입 예정.
2. **SVG 다이어그램 수치 드리프트** — 생성 스크립트가 수수료율 등을 SVG에 하드코딩 → 문서 개정 시 히어로 이미지가 낡은 수치를 계속 노출 가능. 1-6 이후 재생성 필수.
3. **저장소 공개 범위** — 뷰어에서 문서를 빼도 git 이력·public 저장소에는 잔존 → 저장소 비공개 전환 여부 검토.
4. **폼 도구 ↔ 개인정보처리방침 §5 연동** — 외부 폼 채택 시 처리위탁·국외 이전 고지 필수 → 1-3과 묶어 처리.
5. **판매 인프라 미정** — POS·결제 단말·위탁품 재고 관리 도구. 정산 명세와 KPI의 실제 데이터 원천 → 개점 전 결정.
